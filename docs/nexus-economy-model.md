# Nexus Economy Model

Archive-Nexus is the manufacturing event source in the Archive Platform Ecosystem. Its economy model is a synthetic simulation layer used by ArchiveOS Survival Mode to estimate profitability, cash balance, and bankruptcy risk.

This model does not use real customer revenue, real finance data, card numbers, account numbers, or delivery data. Every amount and identifier is synthetic demo data.

## Scope

Nexus records:

- production and shipment revenue generated by Factory A/B/C simulation
- manufacturing costs created by material consumption, maintenance, quality loss, and operation overhead
- service fees billed by Archive-Logistics and Archive-Ledger
- daily profit snapshots for control-tower polling

The economy model is intentionally separate from Outbox routing. A manufacturing event can still be routed to Archive-Logistics or Archive-Ledger, while Nexus also records its own synthetic revenue/cost perspective.

## Profit Formula

```text
Nexus Profit =
  production revenue
  + shipment revenue
  + quality recovery revenue
  + inventory clearance revenue
  - material cost
  - maintenance cost
  - quality loss
  - inventory holding cost
  - logistics fees
  - ledger fees
  - operation cost
```

`cashBalance` is calculated as:

```text
initial synthetic cash + cumulative profit
```

The current initial synthetic cash balance is `10,000,000 KRW`.

## Revenue Events

| Revenue type | Description |
| --- | --- |
| `PRODUCTION_REVENUE_RECOGNIZED` | Synthetic revenue recognized when a production tick creates production output |
| `SHIPMENT_REVENUE_RECOGNIZED` | Synthetic revenue recognized when a shipment is created |
| `QUALITY_RECOVERY_REVENUE_RECOGNIZED` | Reserved for future synthetic recovery revenue |
| `INVENTORY_CLEARANCE_REVENUE_RECOGNIZED` | Reserved for future synthetic clearance revenue |

## Cost Events

| Cost type | Source |
| --- | --- |
| `MATERIAL_COST_INCURRED` | Nexus internal manufacturing simulation |
| `MAINTENANCE_COST_INCURRED` | Nexus internal manufacturing simulation |
| `QUALITY_LOSS_INCURRED` | Nexus internal manufacturing simulation |
| `INVENTORY_HOLDING_COST_INCURRED` | Reserved for future inventory simulation |
| `LOGISTICS_SERVICE_FEE_PAID` | External fee from Archive-Logistics |
| `LOGISTICS_DAILY_SETTLEMENT_FEE_PAID` | External fee from Archive-Logistics |
| `LEDGER_SETTLEMENT_AGENCY_FEE_PAID` | External fee from Archive-Ledger |
| `LEDGER_RECONCILIATION_FEE_PAID` | External fee from Archive-Ledger |
| `NEXUS_OPERATION_COST_INCURRED` | Nexus internal operation overhead |

## Creation Points

| Runtime point | Economy effect |
| --- | --- |
| Production tick | Creates production revenue and operation cost |
| Material consumption | Creates material cost |
| Shipment creation | Creates shipment revenue |
| Maintenance threshold event | Creates maintenance cost |
| Quality defect event | Creates quality loss |
| External fee billing | Creates Logistics or Ledger fee cost |

External fee receipt never creates production revenue and fee events do not create another fee event. This prevents circular revenue/cost loops.

## Bankruptcy Risk

Archive-Nexus exposes a coarse bankruptcy risk signal:

| Risk | Rule |
| --- | --- |
| `LOW` | cash is positive and cumulative profit is non-negative, or runway is healthy |
| `WARNING` | cumulative loss exists and cash runway is low |
| `CRITICAL` | cash balance is non-positive or runway is near exhaustion |

This is a game/simulation indicator for ArchiveOS Survival Mode. It is not financial advice and is not derived from real business accounting.

## APIs

```http
GET /api/nexus-economy/summary
GET /api/nexus-economy/revenue-events
GET /api/nexus-economy/cost-events
GET /api/nexus-economy/profit-snapshots
POST /api/nexus-economy/daily-close?date=YYYY-MM-DD
```

Summary example:

```json
{
  "totalRevenue": 5000000,
  "totalCost": 3200000,
  "profit": 1800000,
  "cashBalance": 11800000,
  "bankruptcyRisk": "LOW",
  "dailyClosePath": "/api/nexus-economy/daily-close?date=YYYY-MM-DD"
}
```

## ArchiveOS Consumption

ArchiveOS can poll:

- `GET /api/nexus-economy/summary`
- `GET /api/nexus-economy/profit-snapshots`
- `GET /api/integrations/summary`

`GET /api/integrations/summary` includes `economySummaryPath` so the control tower can discover the economy endpoint without hard-coding it.
